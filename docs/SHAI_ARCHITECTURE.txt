Shai runtime architecture
=========================

       [USER TERMINAL]
              |
         Ctrl+C ×3 → SIGKILL
              v
       [SHAI RUN WRAPPER]
       (crossterm / pty)
              |
      +-------+-------------------+
      |                           |
[STDIN PROMPT PROXY]       [STDOUT / PTY SNIFFER]
      |  (OSC sequences            |
      |   stripped before          | (balanced JSON scan)
      |   prompt storage)          v
      v                      [BACKGROUND WORKER]
 [CLI AGENT]                       |
 (claude/goose/...)                | (stabilize file, AST summary,
      |                            |  record timeline event)
      |                            v
      +--------------------->[STORAGE ENGINE]
                              (SQLite / redb)


Supporting paths
----------------

1. Agent plugin registry (build.rs + src/agents/):
   At compile time, build.rs scans src/agents/*.rs and generates a static
   GENERATED_REGISTRY of AgentPlugin trait objects. Each agent file exports
   `pub(crate) static PLUGIN: <Type>`. Adding a new agent requires only one
   file — no manual registry edits.

   Each plugin implements:
   - skill_file()      — where to write the context document
   - cmd_setup()       — how to launch the agent CLI with context injected
   - merge_content()   — how to merge shai context with any existing content

   Integration methods in use:
   - flag-based  : `--append-system-prompt-file` (Claude), `--system` (OpenCode)
   - env-based   : GEMINI_SYSTEM_MD, GOOSE_MOIM_MESSAGE_FILE
   - file-based  : AGENTS.md (Kilo), .github/copilot-instructions.md (Copilot)
   - skill-based : --skill-location (Junie)
   - fallback    : .shai/skills/shai-context.md written for manual sharing

2. PTY wrapper (src/cli_commands/run.rs):
   Preserves normal interactive terminal behavior while observing prompts,
   tool payloads, checkpoints, and agent output.
   - STDIN proxy strips OSC terminal-query responses before storing prompts
   - Triple Ctrl+C (within 2 s) sends SIGKILL to the child and its process group

3. PATH guard wrappers (src/cli_commands/guard.rs):
   Prepends wrapper binaries so observable shell commands like `rm`, `git`,
   `mv`, `chmod`, and `chown` can be blocked or snapshotted before execution.

4. JSON adapter layer (src/adapters/):
   Translates agent-specific tool-call JSON shapes into (tool_name, command,
   file_path) tuples for the sniffer. Known agents have dedicated adapters;
   unknown agents fall back to GenericAdapter which covers ~10 common field
   shapes and emits a one-time warning when it sees tool-like JSON it cannot
   classify.

5. Semantic summarization (src/semantic/):
   Turns file snapshots into higher-signal summaries such as
   "Added struct Storage" or "Modified method get_status".
   Languages: Rust, Python, JS/TS, Go, Java, Kotlin, Ruby, Swift, C/C++, Markdown.


Persistent state (.shai/)
-------------------------

timeline.sqlite
  - sessions
  - timeline_events
  - memory_facts
  - memory_decisions
  - memory_refs
  - internal_state

blobs.redb
  - Compressed content blobs (Zstd) indexed by BLAKE3 hash

blobs_archive.redb
  - Archived blobs moved by `shai gc`

project_id
  - Stable project identifier for export/import and shared history
